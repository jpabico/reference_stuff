Some notes for Javascript

Just a few things to keep in mind. I ran into them while reviewing some Javascript.

  • 'Typeof' operator returns a string
  • Set defaults using || (example: a||b ...if a is false, then b)
  • False values: 0, -0, " ", undefined, null, Nan
  • If a property doesn't exist yet but set with the = operator, it is added to the object
  • .indexOf() is a useful method
  • Javascript will not warn you if you are defining a variable with a name that is already taken
  • 'throw' ...keyword used to throw an exception
  • 'try'/'catch' ...keywords used to control exceptions
  • Exceptions move stacks of function calls, removing all the contexts it encounters
  • Every tag of a document is represented by a DOM and can be looked up and interacted with
  • attach (window.open('file_name.html')); to attach to console
  • show (document.body) ...or .body.parentNode ...or .body.chlidNodes ...or.documentElements, etc
  • No two event handlers run at the same time
  • By default, many browsers bubble events up the DOM chain
  • Programs are only allowed to make HTTP requests to the domain that the current page comes from

A neat way of creating variables dynamically!

  • create empty object... var sample_obj = {}
  • use bracket notation to define object properties/attributes...
  • ex: sample_object.name = 'billy' vs. sample_object['name'] = 'billy'
  • create new properties in a for loop... sample_object['name'+i] = new Object();
  • now, sample_object['name1'] (or whatever loop index) is not undefined
  • have an array of values to be assigned to the variables... var names_array = []
  • another loop can be set up so that sample_object['name' + i] = names_array[i]
