API Notes

My notes about APIs from when I first started learning about this stuff...

  • HTTP = HyperText Transfer Protocol. It's a fancy way of saying "rules for getting something from one place to another".
  • Clients make an HTTP request. The server sends back a response.
  • Navigating through a site by clicking links brings you to the next page (representing the next state of the app - a state transition). By following this model of clicking from page to page, you're starting to follow REST (Representational State Transfer) principles(see below).
  • API (application programming interface) specify ways a program can interact with an app (e.g. a program that reads/analyzes data from Twitter uses the Twitter API, which would specify the process for authentication, imporant URLs, classes, methods, etc).
  • API endpoints are locations where particular data are stored (e.g. an API for a video hosting service might have endpoints for the most popular videos, the most recent videos, or videos belonging to a certain category).
  • Some APIs require an API key that grants access and identifies the to the API, which helps the API provider keep track of how their service is used and prevent unauthorized/malicious activity. Some APIs require authentication using a protocol called OAuth.
  • RESTful APIs and web services:
    1. separate the client from the server
    2. do not hold state between requests (i.e. all the info needed to respond to a request is available in each individual request - no data or state is held by the server from request to request)
    3. use http and http methods (4 HTTP Verbs): GET (read), POST (create), PUT (update), and DELETE (destroy)
  • The Request...
    1. request line - tells the server what kind of request is being sent (GET, POST, etc) and what resource it's looking for
    2. header - additional info (e.g. which client is making the request)
    3. body - empty (GET request) or contains data (POST, PUT)
  • The Response...
    1. response line - includes 3-digit HTTP status code
    2. header - includes info about the server and its response
    3. body - contains the text of the response
  • As long as the structure of the API's response is documented, other people can use the API to get info.
  • Response data needs to be parsed (analyzed). Often sent as XML or JSON.
